# 人民城轨2.0--运营后台最终功能清单

## APP

### 算法
1.  **推荐算法**：综合行为分析结果，汇总得出用户喜欢的标签类型得分，针对用户观看视频的推荐算法，依据：用户行为+作品权重+临时推荐。
2.  **用户行为分析**：完播率、停留时间、点赞、评论、搜索、关注、推荐，发布。
3.  **视频预加载机制**：降低视频加载时间，提升体验：推荐接口。

### 关注
1.  **关注作品列表**：关注模块仅展示已关注用户所发布的视频内容，视频展示依照关注用户发布视频时间进行排序展示，优先展示未看的最新发布，其次展示已看视频，按照时间顺序排序。
2.  **关注、取消关注**：当无关注用户时提示：当前没有关注的人，快去关注吧。

### 协会
1.  **协会视频列表**

### 展会
1.  **展会视频列表**
2.  **展位查询**：展位信息导览。
3.  **论坛直播**：对接微赞，直播频道列表、频道下的直播列表（直播、频道封面）。
4.  **录播回访**：对接微赞，频道列表，频道下的视频列表（频道封面、视频）。
5.  **直播用户数据对接**：在直播时判断用户状态，登录后可以使用直播功能。

### 交互
- 所有视频播放模块需支持：关注、点赞、分享、推荐、评论（当前暂定用户发布的评论均需要经过AI合规性审查）。
- 这次保留基础对话智能体（混凝土）查看历史对话、创建新对话。

### 发布
- 图文内容发布（拍摄、本地上传），支持图片轮播时间设置。
- 视频内容发布（拍摄、本地上传）。
- 内容设置（需支持选择发布板块，【协会发布】、【行业发布】、【展会发布】、【内容发布】配合展会板块、协会板块使用）。

### 消息
- **系统消息**：展示系统相关消息，例如：维护通知、系统更新、审核通知（失败、通过）等。

### 我的
- **个人信息**：基础信息。
- **系统设置**：反馈、账号安全（手机号变更、密码修改）、关于我们、退出登录等。
- **注册登录**：短信、密码、用户信息初始化（喜欢类型标签选择，5个及更多）。
- **作品管理**：我的作品、我的推荐、我的点赞、展会视频。

### 搜索
- **搜索服务**：微服务搭建、内容存储（作品内容、标签、用户信息）。
- **搜索内容**：视频、图文、用户。
- **作品搜索逻辑**：作品标签、作品描述、用户描述。
- **搜索标签**：综合、视频、图文、用户（灵活标签，根据匹配度优先展示）。
- **点击某一条时**，展示搜索列表作品内容，点击左上角返回，回到来源页面。

## 运营平台

### AI审核
- **图文审核**：检测用户发布图文中的敏感词（如政治、色情、广告违规词汇）以及图片中的违禁元素（如不良场景、水印）。（对接第三方图文检测服务，提供实时检测接口和记录）。
- **视频审核**：支持视频的抽帧画面检测、音频中的违规语音识别、字幕文本的语义分析。（对接第三方音视频检测服务，异步检测+检测记录、下架视频）。
- **直播审核**：对平台内的实时直播内容进行多维度的监控与审核，包括实时画面监控、音频内容监控、互动内容审核、直播间状态管理。
- **互动审核**：自动检测用户评论、私信和分享文案中的违规内容。
- **协会发布审核**：对接钉钉审批流程（审核流程状态记录）。
- **会展视频内容审核**：发布到展会模块的视频需要经过会展公司进行审核-人工审核。审核固定推到会展公司某一岗位，审核通过后的视频才可正式发布，APP内利用推送完成审核。

### 用户行为统计
- **基础行为数据采集**:
    1.  页面浏览记录（首页、关注页、企业宣传页等访问轨迹）。
    2.  交互行为（点赞、评论、分享、收藏、关注操作日志）。
    3.  发布行为（图文/视频发布时间、频率、内容类型）。（涉及与用户行为分析服务相关功能）。
    4.  视频播放数据（播放时长、完播率）。
    5.  搜索行为（关键词、搜索频次）。

### 作品浏览、点赞、评论、推荐等数据统计
- **内容效果指标**（涉及视频统计相关计算）:
    1.  基础数据：点赞量、评论量。
    2.  视频数据：完播率。
- **数据可视化**:
    1.  核心看板：展示热门作品及当日统计。（按照标签分类统计）。
    2.  分模块展示：展示内容分布及统计。（按照板块分类统计（行业用户、展会用户、协会用户））。

### 用户画像分析
- **基础属性标签**：企业所属行业、企业规模、认证状态等静态信息标签生成。
- **行为标签**：根据用户长期行为数据动态生成，包括高频访问模块、内容偏好、活跃时段等。
- **智能分群**：基于用户画像进行用户分群管理，支持精准营销和个性化服务。
- **标签体系管理**：提供完整的用户标签管理功能，支持标签的创建、编辑、删除和权重调整。

### 平台流量分配配置
- **推荐算法配置**:
    1.  权重因子调整：配置推荐权重及用户标签影响。
    2.  分模块规则：不同模块的推荐逻辑配置。

### 日活、月活等统计
- **核心指标**:
    1.  用户活跃度：日活、周活、月活统计。
    2.  行为活跃度：使用时长。

### 用户管理
- **信息与权限管理**:
    - 基础信息管理：用户企业信息查看与管理。
    - 权限分级：不同用户的权限配置。
    - 用户身份（用户、展会、协会）利用导入方式实现。
- **组织结构**：支持协会内部组织结构划分
    1.  分组管理：用户、参展单位、协会。
    2.  协会内部：根据部门进行区分。
- **账号状态管理**:
    - 账号控制：包括违账号封禁和异常登录检测。
    - 行为管理：记录违规历史与高风险用户管理。

### 消息管理
- **APP系统消息推送**：
    1.  消息推送管理：支持系统消息、审核消息、活动消息、安全提醒等多类型消息推送。
    2.  推送目标设置：支持全员推送、用户分群推送、单用户推送等多种推送方式。
    3.  推送数据统计：统计推送量、到达率、阅读率、点击率等关键指标。
- **消息模板管理**：
    1.  模板创建编辑：支持创建和编辑消息模板，包含变量替换功能。
    2.  模板分类管理：按消息类型对模板进行分类管理。
    3.  模板使用统计：统计各模板的使用频次和效果。
- **定时推送功能**：
    1.  定时发送：支持设置定时发送时间，实现消息的延时推送。
    2.  推送计划管理：管理所有定时推送任务的执行状态。

### 展位管理
- **场馆信息管理**：
    1.  场馆基础信息：场馆名称、地址、描述等基本信息维护。
    2.  场馆状态管理：场馆使用状态的设置和管理。
- **楼层分区配置**：
    1.  楼层信息管理：楼层名称、描述、展位数量等信息管理。
    2.  分区划分设置：各楼层内分区的划分和配置。
    3.  展位图管理：楼层展位图的上传和管理。
- **参展公司数据管理**：
    1.  公司基础信息：参展公司名称、描述、联系方式等信息管理。
    2.  展位分配管理：为参展公司分配具体的展位位置。
    3.  公司状态管理：参展公司的确认状态、审核状态等管理。
    4.  APP账号关联：将参展公司与APP用户账号进行关联。

### 运营数据管理
1.  **资源使用监控**:
    - 实时查看服务器的内存使用率。
    - 实时查看存储空间的占用情况。
    - 实时查看应用的带宽使用情况。
2.  **资源消耗统计**:
    - 显示每天、每周、每月的资源消耗统计。
    - 提供趋势分析功能，识别资源使用的高峰期。
    - 统计每个用户的作品量、资源占用量等数据。
3.  **警报和阈值设置**:
    - 设置内存、存储和带宽的使用阈值。
    - 当资源使用超出设定阈值时，自动触发警报。
4.  **资源使用明细**:
    - 按模块查看详细的资源使用信息。
    - 提供用户级别的详细资源使用报告。
5.  **企业级数据管理**：
    - 数据导入导出：支持用户数据、内容数据的批量导入和导出。
    - 数据备份恢复：提供完整的数据备份和恢复机制。
    - 数据清理维护：定期数据清理和数据库优化功能。

### 直播管理
1.  **频道管理**：维护APP中展示的直播频道列表，包括频道名称、封面、简介等。
2.  **直播管理**：管理特定频道下的直播活动，包括直播标题、封面、计划开播时间，并能同步"微赞"的直播状态。
3.  **直播回放管理**：管理直播结束后由"微赞"生成的回放视频，支持配置回放视频的标题、封面。
4.  **用户数据对接**：确保在APP内跳转至直播间时，能正确传递用户登录状态。

### 日志管理（等保三级合规）
- **安全审计日志**：
    1.  **后台系统登录登出日志**：记录所有后台管理员的登录时间、登出时间、登录IP、设备信息、登录结果（成功/失败）、失败原因等详细信息。
    2.  **权限变更日志**：记录用户权限的创建、修改、删除操作，包括操作人、操作时间、变更内容、目标用户等信息。
    3.  **系统配置变更日志**：记录系统参数、配置项的所有变更操作，包括变更前后的值、操作人、变更时间、变更原因等。
    4.  **数据访问日志**：记录敏感数据的访问行为，包括用户信息查看、导出操作、批量操作等，确保数据访问可追溯。
    5.  **异常行为日志**：记录异常登录尝试、权限越级访问、批量操作、异地登录等可疑行为。
- **业务操作日志**：
    1.  **内容审核日志**：记录所有内容审核操作，包括审核人员、审核时间、审核结果（通过/拒绝）、拒绝原因、内容ID、AI分析结果等，确保审核过程完全可追溯。
    2.  **内容发布日志**：记录APP端所有内容发布行为，包括发布用户、发布时间、内容类型、内容摘要、发布板块、审核状态变更等信息。
    3.  **用户管理日志**：记录用户账号的创建、封禁、解封、权限调整等操作，包括操作人、操作时间、操作原因、目标用户等信息。
    4.  **消息推送日志**：记录所有系统消息推送操作，包括推送内容、目标用户、推送时间、推送结果、操作人员等信息。
    5.  **数据导入导出日志**：记录所有数据导入导出操作，包括操作人、操作时间、数据类型、数据量、操作结果等。
- **系统运行日志**：
    1.  **系统性能日志**：记录系统关键性能指标，包括CPU使用率、内存使用率、存储使用率、网络带宽等运行状态。
    2.  **API调用日志**：记录第三方API调用情况，包括调用时间、接口名称、请求参数、响应结果、耗时等信息。
    3.  **数据备份恢复日志**：记录数据备份和恢复操作的完整过程，包括备份时间、数据范围、备份结果、恢复操作等。
    4.  **系统错误日志**：记录系统运行过程中的错误信息，包括错误时间、错误类型、错误详情、影响范围等。
- **日志管理功能**：
    1.  **日志查询检索**：提供多维度日志查询功能，支持按时间范围、操作人、日志类型、关键词等条件进行精确检索。
    2.  **日志统计分析**：提供日志统计报表，包括操作频次统计、异常行为统计、系统使用趋势分析等。
    3.  **日志完整性保护**：采用数字签名、哈希校验等技术确保日志数据不被篡改，满足等保三级完整性要求。
    4.  **日志存储管理**：日志数据至少保存6个月，重要日志保存1年以上，提供自动归档和清理机制。
    5.  **日志访问控制**：严格控制日志访问权限，仅授权的安全管理员可查看敏感日志，所有日志访问行为均被记录。
    6.  **日志备份恢复**：定期备份日志数据，支持日志数据的快速恢复，确保审计证据的持续性。
    7.  **日志告警机制**：对异常日志事件设置告警规则，包括批量操作告警、异常登录告警、权限变更告警等。
